## 2. 不足コンポーネント詳細分析

### Phase 4: ナビゲーションコンポーネント（0/5 未実装）

#### 4-1. Tabs （優先度: 高）

**用途**:
- コンテンツの切り替え
- 設定画面のセクション分け
- ダッシュボードのビュー切り替え

**必要な機能**:
- **バリアント**:
  - `default` - 通常のタブ（下線）
  - `pills` - 丸みを帯びたタブ
  - `boxed` - ボックス型タブ
- **配置**: `horizontal`, `vertical`
- **サイズ**: `sm`, `md`, `lg`
- **状態**: `active`, `disabled`
- **イベント**: `tab-change` (タブ変更時)
- **属性**:
  - `value` - 現在のアクティブタブ
  - `full-width` - 全幅表示
- **スロット**:
  - `tab-{n}` - 各タブのラベル
  - `panel-{n}` - 各タブパネルのコンテンツ
- **アクセシビリティ**: ARIA tablist/tab/tabpanel

**実装の複雑度**: 中

#### 4-2. Menu / Dropdown（優先度: 高）

**用途**:
- ドロップダウンメニュー
- コンテキストメニュー
- ナビゲーションメニュー
- ユーザープロファイルメニュー

**必要な機能**:
- **コンポーネント**:
  - `<ha-menu>` - メニューコンテナ
  - `<ha-menu-item>` - メニュー項目
  - `<ha-menu-divider>` - 区切り線
  - `<ha-menu-group>` - グループ化
- **トリガー**: ボタン、リンク、カスタム要素
- **配置**: `bottom-start`, `bottom-end`, `top-start`, `top-end`, `left`, `right`
- **状態**: `open`, `disabled`
- **機能**:
  - キーボードナビゲーション（矢印キー）
  - サブメニュー対応
  - アイコン、ショートカット表示
  - チェックボックス/ラジオ項目
- **イベント**: `menu-open`, `menu-close`, `menu-select`
- **アクセシビリティ**: ARIA menu/menuitem

**実装の複雑度**: 高（ポジショニング、フォーカス管理が複雑）

**推奨ライブラリ**: Floating UI (旧Popper.js) でポジショニング

#### 4-3. Breadcrumb（優先度: 中）

**用途**:
- ページ階層の表示
- ナビゲーション補助

**必要な機能**:
- **コンポーネント**:
  - `<ha-breadcrumb>` - コンテナ
  - `<ha-breadcrumb-item>` - 各項目
- **属性**:
  - `separator` - 区切り文字（デフォルト: `/`）
  - `href` - リンク先
  - `current` - 現在のページ
- **スロット**:
  - `separator` - カスタム区切り
  - デフォルトスロット - 項目内容
- **アクセシビリティ**: ARIA breadcrumb

**実装の複雑度**: 低

#### 4-4. Pagination（優先度: 中）

**用途**:
- テーブルのページネーション
- リストのページ分割
- 検索結果の分割

**必要な機能**:
- **属性**:
  - `current-page` - 現在のページ
  - `total-pages` - 総ページ数
  - `page-size` - 1ページの項目数
  - `total-items` - 総項目数
  - `show-size-selector` - ページサイズ選択表示
  - `show-jump-to-page` - ページジャンプ入力表示
- **サイズ**: `sm`, `md`, `lg`
- **バリアント**: `simple`, `full`
- **イベント**: `page-change`, `size-change`
- **機能**:
  - 前へ/次へボタン
  - 最初/最後へボタン
  - ページ番号表示（省略表示対応）
  - ページサイズ変更
- **アクセシビリティ**: ARIA navigation

**実装の複雑度**: 中

#### 4-5. Sidebar / Drawer（優先度: 中）

**用途**:
- サイドナビゲーション
- 設定パネル
- フィルター/サーチパネル
- モバイルメニュー

**必要な機能**:
- **配置**: `left`, `right`, `top`, `bottom`
- **サイズ**:
  - `sm` (256px)
  - `md` (320px)
  - `lg` (400px)
  - カスタムサイズ
- **状態**: `open`, `collapsed`, `fixed`, `overlay`
- **属性**:
  - `open` - 開閉状態
  - `placement` - 配置
  - `width` / `height` - サイズ
  - `overlay` - オーバーレイ表示
  - `backdrop` - 背景クリックで閉じる
- **イベント**: `drawer-open`, `drawer-close`
- **アニメーション**: スライドイン/アウト
- **アクセシビリティ**: ARIA complementary/dialog

**実装の複雑度**: 中〜高

### Phase 5: フィードバックコンポーネント（0/6 未実装）

#### 5-1. Alert（優先度: 高）

**用途**:
- 情報/警告/エラーメッセージ表示
- 成功/失敗フィードバック
- インラインメッセージ

**必要な機能**:
- **バリアント**:
  - `info` - 情報（青）
  - `success` - 成功（緑）
  - `warning` - 警告（黄）
  - `error` / `danger` - エラー（赤）
- **スタイル**:
  - `filled` - 塗りつぶし
  - `outlined` - アウトライン
  - `soft` - 柔らかい背景
- **機能**:
  - アイコン表示
  - 閉じるボタン（dismissible）
  - アクションボタン
- **属性**:
  - `variant` - バリアント
  - `closable` - 閉じる可能
  - `title` - タイトル
- **スロット**:
  - `icon` - カスタムアイコン
  - `title` - タイトルコンテンツ
  - デフォルト - メッセージ本文
  - `actions` - アクションボタン
- **イベント**: `alert-close`
- **アクセシビリティ**: ARIA alert/status

**実装の複雑度**: 低〜中

#### 5-2. Toast / Notification（優先度: 高）

**用途**:
- 非侵入型の通知
- 成功/失敗メッセージ
- システムメッセージ

**必要な機能**:
- **コンポーネント**:
  - `<ha-toast>` - 個別のトースト
  - `<ha-toast-container>` - トーストコンテナ
- **バリアント**: `info`, `success`, `warning`, `error`
- **配置**:
  - `top-left`, `top-center`, `top-right`
  - `bottom-left`, `bottom-center`, `bottom-right`
- **属性**:
  - `duration` - 表示時間（ms、0で自動消去なし）
  - `closable` - 閉じるボタン
  - `title` - タイトル
  - `message` - メッセージ
- **機能**:
  - プログラマティックAPI（`toast.show()`, `toast.success()`等）
  - スタック管理
  - アニメーション（スライドイン/フェード）
  - プログレスバー
- **イベント**: `toast-show`, `toast-close`
- **アクセシビリティ**: ARIA alert/status

**実装の複雑度**: 中〜高

#### 5-3. Modal / Dialog（優先度: 高）

**用途**:
- 確認ダイアログ
- フォーム入力ダイアログ
- 詳細表示モーダル
- 画像プレビュー

**必要な機能**:
- **サイズ**:
  - `sm` (400px)
  - `md` (600px)
  - `lg` (800px)
  - `xl` (1000px)
  - `full` (画面いっぱい)
- **属性**:
  - `open` - 開閉状態
  - `size` - サイズ
  - `closable` - 閉じるボタン表示
  - `backdrop-close` - 背景クリックで閉じる
  - `header-title` - ヘッダータイトル
- **スロット**:
  - `header` - ヘッダーコンテンツ
  - デフォルト - ボディコンテンツ
  - `footer` - フッターコンテンツ（アクションボタン等）
- **機能**:
  - フォーカストラップ（モーダル内でTab移動）
  - Escキーで閉じる
  - スクロール制御（body固定）
  - アニメーション（フェード/ズーム）
- **イベント**: `modal-open`, `modal-close`, `modal-confirm`, `modal-cancel`
- **アクセシビリティ**: ARIA dialog/modal, フォーカス管理

**実装の複雑度**: 高

#### 5-4. Tooltip（優先度: 中）

**用途**:
- ヘルプテキスト
- 補足説明
- アイコンの説明

**必要な機能**:
- **配置**: `top`, `bottom`, `left`, `right`
- **トリガー**: `hover`, `focus`, `click`
- **属性**:
  - `content` - ツールチップテキスト
  - `placement` - 配置
  - `trigger` - トリガー方法
  - `delay` - 表示遅延（ms）
  - `max-width` - 最大幅
- **スロット**:
  - デフォルト - トリガー要素
  - `content` - ツールチップコンテンツ
- **機能**:
  - ポジショニング（Floating UI推奨）
  - 矢印表示
- **アクセシビリティ**: ARIA tooltip

**実装の複雑度**: 中（ポジショニングライブラリ使用推奨）

#### 5-5. Progress / Spinner（優先度: 中）

**用途**:
- ローディング表示
- 処理進捗表示
- ファイルアップロード進捗

**必要なコンポーネント**:

##### A. Progress Bar (`<ha-progress>`)
- **属性**:
  - `value` - 現在値
  - `max` - 最大値
  - `variant` - `default`, `striped`, `animated`
  - `color` - カラーバリアント（`primary`, `success`, `warning`, `error`）
  - `size` - `sm`, `md`, `lg`
  - `show-label` - パーセント表示
- **スロット**:
  - `label` - カスタムラベル
- **アクセシビリティ**: ARIA progressbar

##### B. Circular Progress (`<ha-circular-progress>`)
- **属性**:
  - `value` - 現在値
  - `max` - 最大値
  - `indeterminate` - 不確定状態
  - `size` - `sm`, `md`, `lg`
  - `thickness` - 太さ
- **アクセシビリティ**: ARIA progressbar

##### C. Spinner (`<ha-spinner>`)
- **属性**:
  - `size` - `xs`, `sm`, `md`, `lg`, `xl`
  - `color` - カラーバリアント
  - `speed` - 回転速度
- **バリアント**:
  - `circular` - 円形
  - `dots` - ドット型
  - `pulse` - パルス型
- **アクセシビリティ**: ARIA alert/status

**実装の複雑度**: 低〜中

## 3. 優先度マトリクス

### 高優先度（すぐに実装すべき）
1. **Alert** - フィードバックの基本、実装が簡単
2. **Card** - レイアウトの基本、多用される
3. **Badge** - 小さいが重要、実装が簡単
4. **Tabs** - ナビゲーションの基本
5. **Menu/Dropdown** - インタラクションの基本（複雑だが重要）
6. **Modal/Dialog** - フィードバックの基本（複雑だが重要）
7. **Toast/Notification** - フィードバックの基本（中程度の複雑さ）
8. **Table（基本版）** - データ表示の基本

### 中優先度（次のフェーズで実装）
9. **Progress/Spinner** - ローディング表示
10. **Tooltip** - UXの向上
11. **Avatar** - ユーザー関連UI
12. **Breadcrumb** - ナビゲーション補助
13. **Pagination** - データ表示の補助
14. **Accordion** - コンテンツ整理
15. **Sidebar/Drawer** - レイアウト

### 低優先度（後で検討）
16. **List** - 基本的なHTML要素で代替可能
17. **Skeleton Loader** - UX向上だが必須ではない
18. **DataGrid（高度版）** - 非常に複雑、既存ライブラリ検討

## 4. 推奨実装順序

### Phase 4: 基本フィードバック・表示コンポーネント
**目標**: フィードバックとデータ表示の基本を整える

1. **Alert** - 簡単、基礎的
2. **Badge** - 簡単、頻繁に使用
3. **Card** - 中程度、頻繁に使用
4. **Progress/Spinner** - 中程度、ローディング表示

**期待される成果物**:

## 5. 技術的考慮事項

### 5-1. 必要な外部依存

#### Floating UI
**用途**: Tooltip, Menu/Dropdown, Popover等のポジショニング

**理由**:
- 自前でポジショニングロジックを実装するのは複雑
- 画面端での位置調整、矢印表示等を自動処理
- 軽量で高性能

**対象コンポーネント**:
- Menu/Dropdown
- Tooltip
- (将来的に) Popover, DatePicker等

**インストール**:
```bash
pnpm add @floating-ui/dom
```

#### その他
- **アイコンライブラリ**: 未定（ユーザーが選択？デフォルトアイコンを提供？）
- **日付ライブラリ**: DatePicker実装時に検討（Day.js, date-fns等）

### 5-2. アニメーション戦略

**使用技術**:
- CSS Transitions（シンプルなアニメーション）
- CSS Animations（複雑なアニメーション）
- Web Animations API（プログラマティック制御が必要な場合）

**デザイントークン**:
- `transition.duration.*` - 継続時間
- `transition.timing.*` - イージング関数

**対象コンポーネント**:
- Modal/Dialog - フェードイン/ズーム
- Toast - スライドイン/フェードイン
- Drawer - スライドイン/アウト
- Accordion - スライドダウン/アップ
- Tabs - スライド/フェード

### 5-3. プログラマティックAPI設計

**対象コンポーネント**:
- Toast/Notification
- Modal/Dialog（オプション）

**API例**:
```javascript
// Toast
import { toast } from '@hidearea-design/core';

toast.success('保存しました!');
toast.error('エラーが発生しました', { duration: 5000 });
toast.info('お知らせ', {
  description: '詳細メッセージ',
  action: { label: '確認', onClick: () => {} }
});

// Modal（オプション）
import { modal } from '@hidearea-design/core';

const result = await modal.confirm({
  title: '削除の確認',
  message: '本当に削除しますか?',
  confirmText: '削除',
  cancelText: 'キャンセル'
});

if (result) {
  // 削除処理
}
```

### 5-4. フォーカス管理

**必要な機能**:
- **Focus Trap**: Modal/Drawer内でフォーカスをトラップ
- **Focus Return**: Modal/Drawer閉じる時、元の要素にフォーカスを戻す
- **Initial Focus**: Modal/Drawer開く時、最初にフォーカスする要素

**実装方法**:
- カスタム実装
- または、既存ライブラリ（focus-trap）を検討

**対象コンポーネント**:
- Modal/Dialog
- Drawer/Sidebar（オーバーレイモード時）
- Menu/Dropdown
